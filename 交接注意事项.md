## 一、这是什么 / 怎么读

这是一个 Windows 桌面软件（多 Agent 协同工具），详细功能见 `开发清单.md`。

阅读顺序建议：
1. 先读本文（`交接注意事项.md`）——项目地图、怎么跑、别传什么、还剩什么。
2. 再读 `开发清单.md`——软件是什么、功能全景、技术栈、时间线、待办。
3. 需要继续写代码时，再进 `app/` 看源码。
4. `docs/` 里有辅助文档（审计待办、错误总结、模块化拆分规范），按需查阅。

## 二、怎么跑起来

**基础环境**
- Node.js：npm 11.x，较新的 LTS/Current 版本即可。
- Rust：用 rustup 装稳定版（`Cargo.toml` 最低 `rust-version = "1.77.2"`）。
- Python：3.12 系列，依赖见 `python/requirements.txt`。
- Windows WebView2 Runtime：Tauri 桌面应用需要。

**日常启动（开发）**
进入 `app/` 后：
```bash
npm install
npm run tauri:dev
```

或者直接双击根目录的：
- `环境配置器.bat`：第一次用，自动检测并装 Python / Node / Rust 等环境（调用 `setup-dev-env.ps1`）。
- `启动应用.bat`：环境就绪后日常启动，等价于进 `app/` 跑 `npm run tauri:dev`。

> 这两个 .bat 是 GBK 编码 + CRLF 换行，专门适配中文 Windows 的 cmd（默认代码页 936）。**不要用 UTF-8 重存，也不要在 `if (...)` 代码块里写半角括号 `()`**，否则会报「此时不应有」并且中文乱码。

**迁移后建议至少跑一次自检**
```bash
npm run lint
npm run test -- --run
npm run build
```

## 五、继续开发的关键约束

- **低耦合**：新功能尽量放独立文件 / 目录，主文件只做接线。
- **运行用只读快照**，不把运行状态写回原画布。
- **失败隔离**：某节点失败只影响其下游，不阻断无依赖的并行分支。
- **报告中心只采集已结束运行**，运行中的半成品不提前进统计。
- **总 Agent 会话隔离**：一个会话的发送 / 取消不影响其他会话。
- **姬子系统提示词是文件导入模式**，不要擅自恢复手动编辑。
- **prompt caching 靠稳定前缀**：`chat()` 的 `system` 只放人格块（每轮字节不变才命中缓存）；当前时间、记忆、搜索、摘要等每轮会变的内容放 `text`，不要塞进 `system`。
- **姬子 LLM 调用必须传 scene**：`ChatParams.scene` 必填，新增姬子调用点要同时在 `tokenStatsStore.ts` 的 `JiziScene` 加场景值、`ReportCenter/metrics.ts` 的 `JIZI_SCENE_LABELS` 补中文标签，否则按场景统计漏。
- **新增外部 HTTP 调用**要同步 Tauri plugin-http、Rust 插件注册、capabilities 白名单，不能用通配符放行。
- **安全红线**：生成的自定义工具必须人工审核后才落盘执行，不弱化 AST 顶层副作用扫描。

## 七、正式安装包构建步骤（NSIS）

产出标准 Windows 安装包（NSIS，currentUser 安装），随包带一份可重定位的独立 CPython，目标机无需预装 Python。

**为什么这样设计**
- **currentUser 安装**：装到 `%LOCALAPPDATA%\Programs\multi-agent-tool`（类似 VS Code），无 UAC、安装目录用户可写。应用是「绿色便携」架构，`data/`（含明文 Key）、`outputs/`、`logs/`、`python/` 都在 exe 同级且运行时要写（姬子往 `python/tools/custom/` 写工具、`installer.py` 用 pip 装依赖）。装到只读的 Program Files 会破坏这些功能。
- **随包带 Python**：开发用的 `python/venv/` 的 `pyvenv.cfg` 硬编码开发机绝对路径，拷到别的机器就失效；姬子动态造工具又依赖运行时 `pip install`，PyInstaller 冻结会破坏它。所以用 python-build-standalone（可重定位、自带 pip 的完整 CPython 3.12）。

**构建步骤（需联网，耗时几分钟）**
```bash
# 1. 生成可分发 Python 环境（下载 standalone CPython + pip 装依赖，产出 app/src-tauri/python-dist/）
cd /c/Users/Admin/Desktop/agent-collab
node python/build-dist.mjs

# 2. 构建安装包（前端 build + Tauri 打包，python-dist 作为资源打进包）
cd app
npm run tauri:build

# 3. 产物位置
#    app/src-tauri/target/release/bundle/nsis/multi-agent-tool_<版本>_x64-setup.exe
```

> 改了源码 Python 后、只需要把新代码同步进已有的 python-dist，可跑 `node python/build-dist.mjs --sync-only`（跳过 CPython 下载和 pip）。

**落点校验（安装后）**
安装目录内应有：`python/runtime/python.exe`、`python/app.py`、`python/tools/`。运行时 `python_manager.rs` 的 `find_python_exe()` release 分支按 `python/runtime/python.exe` 定位。

**注意**
- **未签名**：无代码签名证书，首次运行可能触发 Windows SmartScreen「未知发布者」，需点「仍要运行」。
- **体积**：NSIS 包约 120–180MB（standalone CPython + PyMuPDF/pdfplumber/pillow/cryptography 等重依赖）。这是「离线开箱即用 + 保留运行时 pip 造工具」的代价。
- **构建机需联网**：`build-dist` 要从 GitHub 下载 standalone Python；下载后离线可重复 `tauri:build`。
- **Python 版本**：standalone 取 3.12，对齐开发 venv，避免 site-packages ABI 不匹配。

**安装向导界面定制（方案 B：品牌蓝）**
- 相关文件（都在 `app/src-tauri/`）：
  - `license.txt` — 安装时展示的许可协议正文（UTF-8 带 BOM + CRLF，中文才不乱码）。
  - `installer/sidebar.bmp`（164×314）、`installer/header.bmp`（150×57）— 品牌蓝渐变图，欢迎/完成页大图 + 内页横幅。均为 24 位 BMP（NSIS 不吃 PNG，也别存 32 位带 alpha，否则可能显示黑块）。生成脚本见 `.aimana/cowork/2026-07-10/27/assets.ps1`。
  - `installer/installer.nsi` — **覆盖用的自定义 NSIS 模板**，从官方模板 `tauri-cli-v2.11.4` 复制，只加了三处：license 页 `MUI_LICENSEPAGE_CHECKBOX` + 中文勾选文案（不勾「我接受」灰置）、`MUI_BGCOLOR=FFFFFF` / `MUI_TEXTCOLOR=222222`（白底扁平）。其余含 `{{#each resources}}` 等 handlebars 循环原样保留，动不得。
- `tauri.conf.json` → `bundle.licenseFile` 指向 license.txt；`bundle.windows.nsis` 加了 `template` / `installerIcon` / `sidebarImage` / `headerImage`。
- ⚠️ **升级 Tauri 后必看**：`installer.nsi` 是手工维护的模板副本，锁定在 CLI 2.11.4。若升级 tauri-cli，需重新从 `https://raw.githubusercontent.com/tauri-apps/tauri/<新tag>/crates/tauri-bundler/src/bundle/windows/nsis/installer.nsi` 拉新模板，再把上述三处改动重新打上去；否则可能与新 bundler 的变量/资源循环不匹配导致打包失败或漏文件。若嫌维护麻烦，可删掉 `template` 字段回退官方模板（会丢掉勾选框强制确认，品牌图仍保留）。
- **能改到什么程度**：NSIS 的窗口边框、底部按钮栏、进度条是 Windows 原生控件，改不动；可定制的只有侧栏图/横幅图/图标/背景色/协议勾选。想要完全自定义的现代安装界面得换安装器技术栈，成本高，暂不做。
