## 一、拉取与启动

```powershell
git clone https://github.com/weichi0326/agent-collab.git
cd agent-collab
git switch fictionist
git pull --ff-only origin fictionist
```

首次在新电脑开发时运行根目录的 `环境配置器.bat`。日常使用 `启动应用.bat`；启动前检查会识别并关闭属于本项目的旧 Vite 进程，但不会终止占用 `5173` 的其他程序。

手动前端验证：

```powershell
cd app
npm.cmd install
npm.cmd test
npm.cmd run lint
npm.cmd run build
npm.cmd run dev -- --host 127.0.0.1
```

浏览器模式使用独立的 `localStorage`，不能代替 Tauri 桌面端对真实本地文件、原生对话框和已存在用户数据的验收。

## 二、当前已经实现

### 2.1 小说作品管理

- 书架按状态和自定义分类筛选。
- 新建、重命名、删除作品，修改分类、状态和自定义封面。
- 新建、重命名、删除、排序卷和章节，并可跨卷移动章节。
- 章节状态管理、正文编辑、预览、手动保存和 `Ctrl+S`。
- 全书搜索覆盖卷名、章节名和正文，并支持正文批量替换。
- 小说可按卷章顺序导出为 TXT。

### 2.2 大纲、设定与时间线

- 大纲分为故事总纲、卷纲和章纲，不把长篇小说压进单一文本。
- 结构化摘要与详细大纲采用桌面双栏布局；每个层级拥有独立详细文本。
- 支持本地大纲直接导入，也可以创建“整理大纲”或“优化大纲”专业任务。
- AI 大纲结果先进入中文审阅界面；JSON、内部 ID 和英文 schema 只用于机器校验，不展示给用户。
- 设定库支持人物、地点、组织、物品和规则的增删改查与批量导入导出。
- 时间线支持背景、已确认和章节事件的增删改查、排序与批量导入导出。

### 2.3 专业 Agent、工作流与画布

- 小说家注册 10 个专业 Agent，并在工作台 Agent 库中以可折叠分组展示，可拖入普通画布。
- 内置工作流包括 `AI 起草`、`续写下一章`、`整理大纲` 和 `优化大纲`。
- 每条内置工作流有可编辑主流程和只读备用流程；备用开关默认关闭，开启后只在主流程失败时额外运行一次。
- 内置模板不能直接运行，必须由小说家创建带作品上下文的专业任务画布。
- 工作台、已保存画布和运行历史会把小说家画布折叠分组；专业运行历史优先显示章节或业务对象名称。
- 从小说家进入工作流画布后，工作台提供返回小说家工作流中心的入口。
- AI 起草和续写结果先进入待确认草稿；用户确认前不会覆盖正式章节。
- 上下文分析和设定检查节点的结果可以回到章节侧栏；工作流未包含对应节点时给出明确空状态。

### 2.4 数据和恢复

- 索引：`multi-agent-fictionist-index`。
- 章节正文：`multi-agent-fictionist-chapter-<chapterId>`。
- 恢复记录：`multi-agent-fictionist-recovery`。
- 专业任务和备用策略使用各自的 Zustand 持久化存储。
- 作品、卷章、正文、设定、时间线和大纲写入均有大小或归属校验；关键跨记录操作使用恢复记录。
- 系统清理小说家数据时同时移除专业任务、关联画布和运行历史，并在界面中说明能力损失。

## 四、不可破坏的约束

1. 小说家不能另建模型运行器，所有 AI 能力继续复用 master 画布。
2. 普通画布没有专业来源时，保存、运行和历史行为必须保持原样。
3. 专业任务使用不可变上下文快照；正式数据只在用户确认后写入。
4. 主流程可编辑，备用流程只读；取消运行不能触发备用流程。
5. 用户可见界面不显示结构化 JSON、内部 ID 或英文结果角色。
6. 浏览器回归不能证明 Tauri 本地数据迁移、原生文件选择和清理已经通过。
7. 不提交 `data/`、`logs/`、`outputs/`、API Key、虚拟环境或构建产物。

## 五、尚未完成

- 小说家还不是可安装、禁用、升级或卸载的 `.mapp` 包；当前是专业包边界下的内置开发实现。
- 整体数据还没有用户可操作的备份与恢复界面。
- 自定义工作流已经可以新建、重命名、复制和删除，但尚未形成完整的专业任务配置与结果回传设计。
- 尚未在配置真实模型的 Tauri 桌面环境完成全部写作和大纲工作流的人工端到端验收。
- `FictionistWorkspace.tsx` 仍然偏大，后续拆分必须保持当前交互和数据边界，不应与新功能混在同一提交。
- 当前明确不做章节版本历史、删除恢复、云同步、多人协作和手机/平板布局。
