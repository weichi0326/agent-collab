# 小说家 UI 验收基线

> 状态：基线清单和渲染测试已完成。自动截图按维护者偏好取消，最终视觉检查由维护者人工完成。

这份清单记录功能接入前已经认可的小说家桌面界面，用于后续拆分组件、接入真实数据和连接 master 画布时判断视觉回归。当前界面仍是早期雏形，不在仓库中维护 15 张固定截图；布局变化由人工按三档窗口检查。

## 视口

- `1280x720`：最低可用窗口，重点检查操作区、导航和正文是否拥挤或重叠。
- `1600x900`：常见系统缩放或非全屏窗口。
- `1920x1080`：原生 Full HD 桌面。

## 覆盖范围

每个视口都检查：

- `library`：书架。
- `chapters`：正文编辑。
- `canon`：设定库。
- `timeline`：时间线。
- `workflows`：工作流。

## 更新规则

- 后续只有经过明确确认的视觉变更才能调整现有布局、颜色、密度和主要文案。
- 功能接入、文件拆分或状态管理重构不能顺带重做界面。
- 涉及布局或样式时，必须同时检查三个视口和五个界面。
- 验收时使用全新浏览器会话，避免个人本地数据影响结果。
- 人工检查不能替代渲染测试、交互验证、Lint 和生产构建。

## 关键交互

在真实浏览器验证：

1. 从书架打开《雾港来信》。
2. 编辑章节正文后出现“有未保存修改”。
3. “续写下一章”可以打开上下文预览弹窗。
4. 可以切换到设定库、时间线和工作流。
5. 控制台没有错误，页面没有横向溢出、按钮裁切或控件重叠。
