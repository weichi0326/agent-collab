# 小说家 UI 验收基线

这份清单记录已经认可的小说家桌面界面，用于后续拆分组件、增加专业能力和提取安装包时判断视觉回归。不在仓库中长期维护固定截图；布局变化由人工按桌面窗口检查。

## 视口

- `1280x720`：最低可用窗口，重点检查操作区、导航和正文是否拥挤或重叠。
- `1600x900`：常见系统缩放或非全屏窗口。
- `1920x1080`：原生 Full HD 桌面。
- `2048x1152`：宽屏布局，重点检查大纲双栏和工作流列表是否过度拉伸。

小说家不面向手机和平板，不为窄屏增加单独布局。

## 覆盖范围

每个视口都检查：

- `library`：书架。
- `chapters`：正文编辑。
- `outline`：故事总纲、卷纲和章纲。
- `canon`：设定库。
- `timeline`：时间线。
- `workflows`：小说家共享工作流中心。

## 更新规则

- 后续只有经过明确确认的视觉变更才能调整现有布局、颜色、密度和主要文案。
- 功能接入、文件拆分或状态管理重构不能顺带重做界面。
- 涉及布局或样式时，必须检查受影响界面在 `1280x720`、`1600x900`、`1920x1080` 和宽屏中的表现。
- 验收时使用全新浏览器会话，避免个人本地数据影响结果。
- 人工检查不能替代渲染测试、交互验证、Lint 和生产构建。

## 关键交互

在真实浏览器验证：

1. 从书架打开《雾港来信》。
2. 编辑章节正文后出现“有未保存修改”，`Ctrl+S` 可以保存。
3. “AI 起草”或“续写下一章”可以打开任务配置并进入真实工作台画布。
4. 大纲页左侧显示结构化摘要，右侧显示独立详细大纲；卷纲和章纲从左侧树切换。
5. 大纲 AI 审阅只显示中文字段和实际卷章名称，不出现 JSON、内部 ID 或代码块。
6. 设定库和时间线可以完成增删改查，工作流中心可以查看主备流程状态。
7. 从工作流画布可以返回小说家工作流中心。
8. 控制台没有错误，页面没有横向溢出、按钮裁切或控件重叠。

## 本轮结果

- 书架、正文、大纲、设定库、时间线和工作流中心均接入真实状态。
- 大纲采用接近等宽的桌面双栏；`2048x1152` 下详细大纲输入区约 718px 宽、760px 高。
- `1280x720` 下结构表单和详细大纲没有横向溢出或相互覆盖。
- 输入详细大纲后会显示“有未保存修改”，证明编辑器仍连接真实草稿状态。
- 使用项目标准开发地址验证时，目标页面控制台为 0 个错误、0 个警告。
