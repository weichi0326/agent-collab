# 姬子半屏 / 全屏切换设计

日期：2026-07-12

## 目标

姬子面板保留现有半屏工作方式，同时允许用户通过右上角滑动开关切换为内容区全屏。全屏只覆盖应用标题栏以下区域，不进入操作系统全屏，不隐藏报告中心、设置、保存、另存为和运行等顶部命令。

## 交互

- 姬子首次使用默认半屏，半屏高度沿用当前 `50vh`。
- 姬子展开后，右上角显示一个二态滑动开关，两个状态分别为“半屏”和“全屏”。
- 开关初始值来自用户上次选择；用户切换后立即生效并持久化，下次启动继续使用该模式。
- 收起姬子只改变展开状态，不重置半屏 / 全屏选择；再次展开时恢复上次模式。
- 开关必须提供清晰的可访问名称、当前状态和 Tooltip，不能只依赖图形或颜色表达状态。
- 收起姬子的现有居中入口保持不变，避免改变用户已形成的操作习惯。

## 布局

### 半屏

- 继续参与现有纵向布局，占用约一半视口高度。
- 下方画布、Agent 侧栏、属性面板和状态栏保持可见。

### 全屏

- 姬子抽屉成为标题栏下方的覆盖层，填满剩余内容区域。
- 顶部标题栏保持可见和可操作。
- 画布、左右侧栏、状态中心及运行组件继续挂载，只在视觉上被覆盖，不触发卸载、重建或运行中止。
- 覆盖层使用稳定层级和纯色背景，不透出下方画布，也不产生双滚动条。
- 会话栏保持固定宽度；聊天正文继续限制舒适阅读宽度，避免超宽文本行。

## 状态与实现边界

- 在 `uiStore` 增加持久化布尔状态 `drawerFullscreen`，默认 `false`。
- `drawerExpanded` 继续保持瞬态，不持久化；`drawerFullscreen` 纳入持久化字段。
- `MasterAgentDrawer` 读取并切换该状态，根据 `expanded && drawerFullscreen` 添加全屏样式类。
- 全屏使用覆盖布局，不通过条件渲染移除工作台组件。
- 不调用 Tauri 或浏览器 Fullscreen API。

## 组件

- 使用 Ant Design `Switch` 表达二元模式，放在姬子抽屉控制条右侧。
- 开关显示“半屏 / 全屏”状态文案，并使用 Tooltip 说明切换结果。
- 开关尺寸固定，控制条预留右侧空间，确保居中的“姬子”入口不会因开关出现而偏移。

## 响应式要求

- `1024 × 640`：全屏内容不溢出标题栏下方区域，输入框始终可见，会话栏与聊天区不重叠。
- `1920 × 1080`：聊天正文保持可读宽度，不被拉伸成超长行，左右留白均衡。
- 半屏和全屏切换过程中不改变消息列表、草稿、附件、当前会话或滚动容器结构。

## 测试与验收

1. Store 测试：默认半屏；切换后状态更新；持久化字段包含 `drawerFullscreen`。
2. 组件测试：开关存在，具备可访问名称，点击后半屏 / 全屏类正确切换，收起再展开仍保持选择。
3. 回归测试：切换模式不会卸载姬子内容或清空会话状态。
4. 视觉检查：使用 `1024 × 640` 和 `1920 × 1080` 截图验证标题栏保留、覆盖范围、输入框位置、会话栏与消息区布局。
5. 工程门禁：前端测试、Lint 和生产构建全部通过。

## 不在本次范围

- 自由拖拽调整姬子高度。
- 操作系统级全屏或隐藏标题栏。
- 修改会话栏宽度、消息样式、输入框功能或 Skill 弹窗布局。
- 为半屏 / 全屏增加额外快捷键。
