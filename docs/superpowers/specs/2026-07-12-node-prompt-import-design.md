# 节点系统提示词导入折叠区设计

日期：2026-07-12

## 目标

将节点属性面板中可直接编辑的“系统提示词”文本框，改造成与姬子人格导入完全一致的本地文件导入与折叠预览交互。

## 交互

- 系统提示词只能通过本地文本文件导入，不再支持手动编辑。
- 支持 `txt`、`md`、`csv`、`json`、`log`、`xml`、`yaml` 和 `yml`。
- 非文本文件显示警告，不修改节点现有提示词。
- 文件读取失败显示错误，不修改节点现有提示词。
- 内容超过 14,000 字符时截断并提示。
- 导入成功后立即写入当前节点的 `systemPrompt` 和来源文件名。
- 显示“当前来源：文件名”；没有来源信息的旧节点显示“早期手动编辑（无关联文件）”。
- 预览默认折叠，折叠行显示“展开系统提示词预览”和字符数。
- 展开后显示只读文本区域，不能手动修改。
- 再次导入文件会替换提示词内容和来源文件名。
- 只读历史节点可以展开预览，但不能触发文件选择。

## 数据

在 `AgentNodeData` 中增加可选字段：

```ts
systemPromptSourceName?: string;
```

字段随当前画布和保存画布持久化。画布导出与导入保留该文件名元数据；它只表示导入来源，不保存真实文件路径，也不重新读取源文件。

## 视觉

- 复用现有 `.master-config-preview`、`.master-config-preview__toggle` 和 `.master-config-preview__summary` 样式。
- 导入按钮、来源文字、折叠箭头、字符数和只读预览与姬子人格导入保持一致。
- 节点属性面板不新增卡片层级，不改变其他属性字段顺序。

## 实施范围

仅修改：

- `app/src/components/PropertiesPanel.tsx`
- `app/src/stores/canvas/types.ts`
- `app/src/lib/canvasTransfer.ts`
- `app/src/App.css`
- 对应测试文件

不修改 Agent 库模板编辑、姬子配置、节点运行、模型调用、工具标签和数据来源逻辑。

## 验收

- 导入有效文本后，节点提示词与来源文件名同步更新。
- 折叠与展开不修改数据。
- 展开预览为只读。
- 旧节点无来源文件名时正常显示兼容文案。
- 画布导出再导入后仍保留来源文件名。
- 只读节点禁用导入操作。
- 原有画布、属性编辑和构建测试全部通过。
