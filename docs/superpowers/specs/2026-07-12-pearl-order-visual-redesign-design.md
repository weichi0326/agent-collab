# 珍珠秩序全局视觉升级设计

日期：2026-07-12

## 目标

将应用统一升级为“珍珠秩序”视觉体系：以低饱和莫兰迪色彩、平整珍珠表面、局部磨砂玻璃、克制留白和柔和动效构成 2026 前沿极简桌面工作界面。

本次只调整软件 UI、界面布局和界面过渡，不改变业务功能、数据结构、状态管理、后端接口、文件系统行为或运行编排逻辑。

成功标准：

- 工作台、设置中心、报告中心、姬子区域和二级弹窗使用同一套颜色、圆角、间距、表面和动效规则。
- 用户能从相同的导航、标题、按钮、表单、列表和弹窗形态识别一致的产品语言。
- 玻璃效果不降低文字、表格、画布节点和配置表单的可读性。
- 页面、抽屉和弹窗切换没有闪屏、突跳或尺寸瞬变。
- 最小支持窗口 `1024 x 640` 下不发生文字、工具条、侧栏和主要操作重叠。

## 视觉方向

采用“局部玻璃、主体平整”的珍珠秩序方案。

- 顶栏、悬浮工具条、下拉菜单、姬子浮层和弹窗可以使用磨砂玻璃。
- 工作区、设置正文、报告正文、列表和属性编辑区使用接近实体的珍珠表面。
- 云雾柔光只作为大面积背景的低对比照明，不使用独立光球、彩色斑点或装饰性渐变块。
- 不把所有区域包装成悬浮卡片，不创建卡片嵌套卡片的结构。
- 不通过牺牲信息密度、对比度或操作效率来表现极简风格。

## 设计令牌

### 色彩

基础色彩采用低饱和、偏冷的莫兰迪体系：

- 应用背景：雾灰珍珠白。
- 主表面：高明度珍珠白，允许少量透明度但不依赖背景模糊维持层次。
- 次级表面：略深的雾灰绿，用于侧栏、分组和弱选中态。
- 主文字：深灰绿，不使用纯黑。
- 次级文字：中性灰绿，保证说明文字仍清晰可读。
- 主强调色：灰鼠尾草绿。
- 辅助色：雾蓝、柔灰紫、灰红和浅金灰，只用于状态、图表或必要区分。
- 边框：低对比冷灰，主要依靠细线和表面明度差建立层级。

禁止使用高饱和品牌蓝作为全局主色，禁止使用大面积单一紫色、深蓝色或米黄色主题。

### 字体

- 使用系统无衬线字体栈，优先保证 Windows 中文显示质量。
- 正文使用常规字重，标题使用中等字重，只在关键数值或状态上使用更强字重。
- 页面标题保持克制，不在紧凑面板内使用超大字号。
- 字间距保持 `0`，不使用负字间距。
- 次要说明不得小于现有可读性下限。

### 圆角

- 图标按钮和紧凑控件：`10px`。
- 输入框、选择器和普通按钮：`12px`。
- 小型面板和画布节点：`16px`。
- 主要容器：`20px`。
- 弹窗、抽屉内面板和大型浮层：`24px`。

胶囊形状只用于标签、状态和分段控制，不用于普通文字按钮和大面积导航项。

### 间距与密度

使用 `4 / 8 / 12 / 16 / 24 / 32 / 48px` 间距序列。

- 页面级区域采用舒适留白。
- 列表、属性编辑、画布工具和高频操作区保持紧凑。
- 所有固定格式元素使用稳定尺寸，加载、悬停和文本变化不得引起布局位移。
- 内容区以对齐、分隔线和留白组织层级，不依赖大量卡片边框。

### 表面与阴影

- 实体表面使用轻微透明度、细边框和低强度漫射阴影。
- 玻璃表面使用 `18-24px` 背景模糊，并提高表面不透明度确保文字清晰。
- 弹窗使用更明显但柔和的大范围阴影，避免硬边和高对比投影。
- 画布节点阴影只用于区分层级，不让每个节点呈现悬浮装饰效果。

## 布局体系

### 应用外壳

- 保留现有顶部系统标题栏，继续承担窗口拖动、最小化、最大化和关闭操作。
- 标题栏下方使用统一应用主体布局。
- 一级入口采用窄侧轨或现有入口的等价紧凑形态，统一工作台、报告和设置的导航位置。
- 一级页面切换不重新创建应用外壳，避免窗口内容整体闪烁。

### 工作台

- 保留资源区、画布区和属性区的核心结构。
- 资源区和属性区采用平整珍珠表面，以浅分隔线建立边界。
- 画布成为视觉主体，减少外围容器边框和重复标题。
- 画布工具条使用局部玻璃浮层，图标按钮尺寸固定。
- 侧栏收起、姬子展开和属性区宽度变化必须使用稳定的布局插值。

### 设置中心

- 保留固定纵向分类导航与平整正文区域。
- 设置项使用标题、说明、控件和分隔线组成，不新增嵌套卡片。
- 双栏管理器只用于模型、工具等确有列表与详情关系的页面。
- 姬子半屏与全屏设置继续位于“姬子配置”对应栏位，不在抽屉重复提供。

### 报告中心

- 与设置中心共享页面标题、导航返回、正文宽度和页面切换动效。
- 指标卡只保留必要的汇总数据；主要报告区域使用平整布局、表格或图形。
- 状态颜色同时配合文字或图标，不只依赖颜色表达含义。

### 姬子区域

- 姬子对话区使用与应用主体一致的珍珠表面。
- 输入区、会话栏和必要悬浮操作可以使用轻量玻璃效果。
- 半屏与全屏之间采用尺寸和位置过渡，不重新挂载会话内容。
- 消息、代码块、工具调用和建议项使用信息层级区分，不堆叠多层卡片。

### 二级界面

- 简短配置和确认操作使用统一居中弹窗。
- 长内容或需要持续参考主页面的配置使用统一右侧面板。
- 不再为不同功能保留彼此割裂的圆角、标题栏、页脚和遮罩样式。
- 弹窗内部不得再打开同尺寸弹窗；需要二级选择时使用局部展开、下拉菜单或尺寸更小的确认框。

## 组件规范

- 按钮：主操作使用低饱和实体强调色；次要操作使用透明或浅表面；纯图标操作使用 Lucide 图标并提供提示。
- 输入控件：统一高度、圆角、边框、聚焦环和错误状态；不通过阴影模拟输入框。
- 导航：激活态使用浅强调背景、清晰文字和必要的细指示线，不使用高饱和整块填充。
- 分段控制：用于密度、模式和视图切换；选中项使用实体珍珠表面和轻微阴影。
- 开关与复选框：只表达二元设置，不用文字按钮替代。
- 列表：依靠行高、对齐和悬停态组织，不为每一行创建独立卡片。
- 弹窗：统一标题区、内容间距、关闭按钮、操作区和最大宽度规则。
- 工具提示：只用于图标含义或补充说明，不隐藏完成任务所需的核心信息。

## 动效规范

统一沿用并调整全局动效令牌：

- 微交互：`140-180ms`。
- 面板展开与局部切换：`220-280ms`。
- 页面、弹窗和大尺寸模式切换：`300-360ms`。
- 标准缓动：柔和自然减速。
- 强调进入：最多 `8-12px` 位移，配合透明度，不使用夸张缩放。

具体行为：

- 工作台、设置中心和报告中心切换时保留应用外壳，正文执行轻微横向或纵向位移与透明度过渡。
- 弹窗打开时遮罩渐入、背景轻微模糊，弹窗由小幅位移进入；关闭执行反向动画。
- 右侧面板从边缘平滑展开，宽度与内容同时过渡，避免先出现内容再改变尺寸。
- 姬子半屏和全屏切换使用容器尺寸插值，聊天滚动位置与输入内容保持不变。
- 悬停只改变表面、边框或轻微位移，不使用持续循环动画。
- `prefers-reduced-motion` 下关闭位移、缩放和模糊动画，只保留必要的即时状态变化。

## 响应式与窗口约束

- 最小支持窗口为 `1024 x 640`。
- 宽度不足时按“次要信息优先收起”的顺序处理：属性区、资源区文字、辅助操作。
- 不通过无限缩小字体、按钮或输入框维持多栏。
- 弹窗高度受可用窗口限制，正文滚动，标题和操作区保持可见。
- 最长中文标签、按钮文字和状态文案不得与图标或相邻控件重叠。

## 明确排除

本次不修改：

- `app/src-tauri/` 下的任何 Rust、存储或权限逻辑。
- `app/src/stores/` 下的状态结构、持久化和业务行为。
- `app/src/lib/` 下的模型、搜索、姬子、工具、文件和编排逻辑。
- `app/src/settings/` 下的草稿、目录、导航和轮询业务逻辑。
- 模型、工具、搜索、画布、Agent 或报告的数据协议。
- 构建工具、依赖版本、包管理配置和开发环境。
- 与视觉升级无关的安全、性能、可访问性或架构重构。

如果视觉实现确实需要修改排除范围中的文件，必须停止当前阶段，说明原因、必要性和替代方案，并获得用户单独批准。

## 验收与验证

每个阶段至少执行：

- 对应组件和交互测试。
- 前端 lint、类型检查和生产构建。
- `1024 x 640`、常规桌面和宽屏三种尺寸的浏览器截图检查。
- 工作台、设置中心、报告中心、姬子半屏与全屏、所有二级弹窗的视觉检查。
- 动画开始、进行中和结束状态检查，确认无重叠、闪屏和布局跳变。
- 深浅背景下的文字、边框、输入控件和状态对比检查。

阶段验收重点不是“所有文件统一重写”，而是该阶段列出的界面是否遵守同一套设计令牌和交互规则。

## 范围控制

- 每次只实施一个阶段。
- 开始阶段前记录允许修改的文件清单。
- 完成阶段后对照 `git diff --name-only` 检查实际修改文件。
- 清单外文件出现修改时立即停止并说明，不将其混入当前阶段。
- 不因发现旧代码问题而顺手修复，除非该问题直接阻止本阶段视觉实现。
- 不在视觉升级中迁移业务逻辑或更改现有功能入口的业务含义。
