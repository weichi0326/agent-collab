# 极小折线自动拉直（拖拽结束时对齐相连节点）

> 日期：2026-07-13
> 状态：待确认
> 关联：orthogonalRoute + edgeRouting（连线端口固定节点中心）

## 一、背景与现象

用户手动把两个上下相连的节点摆到近似竖直对齐后，连线中间出现一个几像素的横向小台阶（Z 形），见用户截图（红箭头指向中段的小拐）。连线端口固定在节点水平中心，两节点中心 x 差几像素时，正交路由必须插入一条极短横段连接上下两条竖线，就是这个小台阶。

用户判断：**这种极小折线没必要存在，应自动拉直，且允许下方节点跟着移动一点点来对齐。**

## 二、根因（已核实）

- 连线端口固定在节点水平中心（`nodePorts.ts` Top/Bottom → 水平中点）。
- 自动路由 `createOrthogonalRoute`（竖直连接）产出 `source →(source.x,中线)→(target.x,中线)→ target` 三段，中段横线长度 = 两节点中心 x 偏移 `|dx|`。
- **只要 `source.x ≠ target.x`，正交路由画不出真正笔直的竖线**——要么中段留台阶，要么端口处留拐。要真正变直，唯一办法是让两节点中心坐标相等，即移动其中一个节点。这与用户"下方节点移动一点"的直觉一致。

## 三、决策（已与用户确认）

在**拖拽结束**时，对被拖动节点的**相连**边做一次"中心对齐拉直"，而非改路由层、也非对画布任意节点做通用对齐。

- **适用方向**：竖直连线移**下方**节点的 x；水平连线移**右侧**节点的 y。对称处理。
- **阈值**：两节点中心几何偏移 `< 8`（flow 坐标，**与缩放无关**）才拉直；≥8 视为用户有意错开，保留台阶。
- **移动对象**：竖直连线里 center.y 较大者（下方）移动；水平连线里 center.x 较大者（右侧）移动。因此：
  - 拖上方节点 → 下方节点跟着移（符合"下方节点移动"）。
  - 拖下方节点 → 下方节点即被拖节点自身，snap 到上方父节点中心（落点即对齐，体验更好）。
- **多父冲突**：同一节点同一轴被 2 条以上边要求移到不同坐标时 → **不动**（顾此失彼，直接跳过该节点该轴）。
- **爆炸半径**：只处理**被拖节点直接相连**的边，不递归传播到下游更远节点。链式对齐由用户逐个拖动完成，保证行为可预测。

### 为什么放在拖拽结束、而不是路由层或每帧

- 路由层无法在中心不等时画出真正的直线（端口固定中心），硬拉直会在端口处留小拐，治标不治本。
- 绝不能在每次渲染里改 `node.position`：会与用户拖拽打架、可能渲染死循环、静默篡改已保存数据。
- `onNodeDragStop` 是离散、用户发起的时机，只动一次，安全可控。

## 四、机制细节

**判定方向**（与 `edgeRouting.ts` 一致）：对一条边的两端中心，`|dx| >= |dy|` 视为水平连接，否则竖直连接。

**竖直连接**：连接横段长度 = `|cxUpper − cxLower|`。若 `0 < 偏移 < 8` → 把下方节点 x 平移 `cxUpper − cxLower`（使下方中心 x = 上方中心 x）。

**水平连接**：连接竖段长度 = `|cyLeft − cyRight|`。若 `0 < 偏移 < 8` → 把右侧节点 y 平移 `cyLeft − cyRight`。

**中心计算**：`center = position + (measured 尺寸)/2`。任一端缺 `measured` 宽/高（尚未测量）→ 跳过该边，不猜尺寸。

**冲突去重**：把所有候选移动按 `nodeId+axis` 分组；同组目标值一致则保留一个，出现不同目标值则整组丢弃（不动）。

**历史/撤销**：`onNodeDragStart` 已 `pushHistory` 记录拖前全量快照（含将被移动的下方节点原位置）。拉直属于本次拖拽收尾，**不再额外 pushHistory**，一次撤销可同时回退拖动与拉直。

**应用方式**：用 React Flow 的 `position` 型 `NodeChange` 经现有 `applyNodes(activeId, changes)` 落地（内部 `applyNodeChanges`）。直接改 store，不经 `onNodesChange`，无回环。

## 六、验证方法

- `npx tsc --noEmit` 通过；`npm run lint`（oxlint）通过；`npm run test -- --run` 全过（含新增 edgeStraighten 用例）。
- 实机（Tauri dev）：
  1. 复现截图场景：上下两节点相连、中心错开几像素 → 拖动其一放开 → 小台阶消失、连线变直、下方节点轻移到位。
  2. 拖下方节点落到父节点正下方偏一点 → 松手自动对齐父中心。
  3. 水平相连两节点近似对齐 → 右侧节点 y 自动对齐、连线变直。
  4. 回归：中心错开 >8px（有意错位）→ 台阶保留，不被强拉。
  5. 回归：下方节点连两个 x 不同的父 → 松手后不乱移。
  6. 撤销一次 → 拖动+拉直一并回退。
  - 若无法起 dev，明确说明"仅静态验证"，不谎报 UI 通过。

## 七、风险

1. **移动了非被拖节点**：拖上方节点时下方节点会动。属用户明确要的行为；仅 <8px 微移，且只动直接相连节点，不递归。
2. **拖下方节点自身被 snap**：落点自动吸到父中心，可能与用户想略微错开冲突——但 <8px 且用户已同意"自动直线"，可接受；>8px 不触发。
3. **与拖拽中对齐吸附叠加**：两者时机不同（吸附在拖动中、拉直在松手后），拉直只针对相连节点+中心，互补不冲突。
4. **缺 measured 时不处理**：极早期未测量的节点不参与，避免猜尺寸导致错移。
