# 统一设置中心设计

日期：2026-07-12

## 目标

将当前分散在标题栏下拉菜单、姬子会话栏和独立弹窗中的系统级配置统一迁入独立设置中心。点击顶部“设置”后进入标题栏下方的二级页面，使用左侧分类导航切换右侧配置内容。

设置中心不迁移 Skill 管理，不迁移画布、Agent、运行或诊断等业务对象操作，也不提供清空全部本地数据功能。

## 页面形态

- 设置中心采用独立二级页面，不使用大型 Modal。
- 应用标题栏始终保留，标题模式显示“设置”。
- 设置视图提供明确的“返回工作台”按钮。
- 左侧为固定分类导航和设置搜索框；右侧为当前配置页。
- 设置页覆盖标题栏以下的工作台，但工作台、画布、姬子和运行组件继续挂载。
- 返回工作台后，画布现场、姬子会话、输入草稿、附件、消息滚动位置、联网搜索开关和运行状态保持不变。
- 右侧内容使用无嵌套卡片的设置行、分隔线和必要的双栏管理器，避免弹窗套弹窗和卡片套卡片。

## 信息架构

```text
AI 能力
  模型服务
  联网搜索

姬子
  姬子配置

扩展
  工具库

系统
  系统与数据
```

首版只有以上五个页面，不创建没有实际配置内容的“常规”“外观”等空分类。

## 导航与入口

### 顶部设置

- 移除当前“设置”按钮的模型 / 搜索 / 工具下拉菜单。
- 点击“设置”直接打开设置中心。
- 在设置视图中，标题栏保留并显示“返回工作台”。
- 顶部“重启后台”继续保留在工作台，作为高频故障处理快捷操作。

### 姬子快捷入口

- 移除姬子会话栏中的齿轮按钮。
- 半屏 / 全屏显示模式开关从姬子抽屉控制条迁入“姬子配置”页。
- 姬子聊天框的 Skill 按钮、Skill 管理页面、导入和复写流程全部保持不变。
- 姬子聊天框的模型下拉选择保持不变。
- 姬子聊天框的联网搜索开关保持不变。

## 页面内容

### 模型服务

迁入现有 `ModelConfigModal` 的全部能力：

- 已配置实例搜索、选择、标星和删除。
- 官方厂商、第三方中转和自定义配置。
- 名称、BaseURL 和 API Key 编辑。
- 保存、更新、通信测试和获取模型列表。
- 手动添加模型。
- 模型别名、启用状态、长上下文、视觉、音频能力标记和能力推断。

页面继续使用左侧厂商 / 配置选择与右侧模型列表的双栏结构。弹窗外壳与内容逻辑拆开，内容面板直接嵌入设置中心。

模型草稿有未保存修改时，切换设置页或返回工作台必须弹出“放弃修改 / 继续编辑”确认，不允许静默丢失。

### 联网搜索

迁入现有 `SearchConfigModal` 的全部能力：

- 多搜索厂商 API Key。
- 厂商启用 / 停用。
- 搜索优先级上移和下移。
- 获取 Key 链接。
- Key 可用性测试与错误反馈。

搜索设置继续直接读写现有 `useSearchStore`，不创建设置中心专用副本。姬子聊天仍通过现有 `hasConfiguredSearch`、`activeSearchEntries` 和 `searchWithFailover` 读取同一状态。

进入设置中心不得卸载 `MasterAgentPanel`，因此聊天框本地的 `webSearchOn`、草稿、附件和正在执行的请求不会被重置。搜索配置修改后，下一次聊天请求立即使用最新厂商状态和顺序。

### 姬子配置

迁入现有 `MasterConfigModal` 的全部能力：

- 本地人格 / 系统提示词文件导入。
- 当前来源和字符数展示。
- 人格文本预览展开 / 收起。
- 恢复默认人格。
- 节点失败时自动诊断开关。
- 展开后的显示模式（半屏 / 全屏），切换立即生效并持久化。
- 长期记忆查看、分类添加、删除和整理。
- 保存后对新消息立即生效。

姬子配置从弹窗改为设置面板。人格草稿有未保存修改时，切换页面或返回工作台需要离开确认。

显示模式使用现有 `drawerFullscreen` 状态，不计入人格草稿，也不触发未保存离开确认。设置中心是该开关的唯一入口，姬子抽屉控制条不再保留重复开关。

Skill 不属于本页，也不出现在设置中心导航或搜索结果中。

### 工具库

迁入现有 `ToolConfigModal` 的全部能力：

- Python 服务状态横幅。
- 内置工具列表和落地状态。
- 自定义工具同步、注册 / 加载错误状态。
- 工具详情、依赖、能力和实现信息。
- 自定义工具删除及确认。

工具页保留左侧列表、右侧详情的双栏布局。仅工具页处于活动状态时轮询 Python 服务和同步工具元数据，离开页签后停止轮询，避免设置中心后台持续请求。

### 系统与数据

首版提供安全的查看、目录入口和修复能力：

- Python 后台运行状态与重启。
- 环境完整性检查和异常说明。
- 应用版本、前端版本、Rust / Tauri 信息和 Python 服务版本。
- 本地数据目录、输出目录和日志目录位置。
- 打开输出目录和日志目录。
- 本地数据类型及占用概览。
- 环境配置器和启动预检的修复指引。

本页不提供“清空全部数据”、批量删除画布、删除模型密钥、删除会话、删除用户 Skill 或删除用户工具。已有顶部“重启后台”继续保留，系统页提供更完整状态而不是取代快捷按钮。

## 状态与组件架构

### 主视图状态

- `AppView` 从 `workspace | reports` 扩展为 `workspace | reports | settings`。
- `settingsSection` 表示五个设置页面之一，并在本次应用会话内记住最后访问页。
- 设置视图不需要跨重启持久化，应用启动仍进入工作台。

### 保持工作台挂载

设置中心作为标题栏下方的覆盖层呈现，不使用 `view === 'settings'` 条件移除工作台节点。

```text
app-shell
  TitleBar
  WorkspaceLayer（始终挂载）
    MasterAgentDrawer
    JiziCommandCenter
    Canvas / Sidebars / Properties
  SettingsCenter（设置状态时覆盖 WorkspaceLayer）
```

覆盖层使用不透明背景和独立滚动，不透出工作台，也不产生页面级双滚动条。`WorkspaceLayer` 在设置状态下不可点击和不可被键盘聚焦，但内部 React 组件、请求控制器和运行状态继续存活。

### 配置面板拆分

现有组件拆为可复用内容面板，Modal 外壳只在确有旧入口需要兼容时保留：

- `ModelSettingsPanel`
- `SearchSettingsPanel`
- `JiziSettingsPanel`
- `ToolSettingsPanel`
- `SystemDataSettingsPanel`

设置中心只负责导航、搜索、页面标题、离开保护和活动面板选择；各面板继续直接使用原有 Store 与后端 API，不复制业务逻辑。

## 设置搜索

- 左侧搜索框只搜索设置页面名称、说明和预定义关键词。
- 搜索结果仍跳转到五个现有页面，不生成临时配置页。
- 搜索关键词包括厂商、API Key、BaseURL、模型、搜索、人格、记忆、诊断、工具、Python、日志、输出和环境。
- Skill 相关关键词不出现在设置中心搜索结果中。

## 保存与错误处理

- 模型和姬子配置保留显式保存，未保存离开必须确认。
- 搜索厂商启用、Key 和优先级继续即时写入 Store。
- 工具删除、后台重启等有副作用操作继续使用现有确认与消息反馈。
- 单个配置页加载失败时在右侧内容区展示错误，不让整个设置中心白屏。
- 返回工作台后不自动关闭或重置姬子聊天框的联网搜索状态。

## 响应式布局

- 桌面宽度 `>= 1200px`：左侧导航约 240px，右侧内容限制可读宽度，模型和工具页可使用更宽双栏区域。
- `1024px` 最小应用宽度：左侧导航缩至约 210px，右侧保持独立滚动；模型和工具双栏不得重叠。
- 不面向手机视口设计，遵循当前 Tauri 最小窗口宽度 `1024px`。

## 测试与验收

1. 导航测试：顶部设置进入二级页，五个分类可切换，返回工作台恢复原现场。
2. 挂载测试：进入设置时画布、姬子会话和运行组件仍在 DOM；返回后草稿、附件和联网搜索开关保持。
3. 联网搜索回归：设置页修改 `useSearchStore` 后，聊天调用继续读取同一状态和优先级。
4. 入口测试：姬子会话齿轮移除；Skill 按钮、模型下拉和顶部重启后台仍存在。
5. 未保存保护：模型和姬子草稿切页 / 返回时触发确认。
6. 工具生命周期：进入工具页开始轮询，离开工具页停止。
7. 系统页安全：不存在清空全部本地数据入口。
8. 视觉检查：`1024 × 640`、`1440 × 900`、`1920 × 1080` 下检查导航、双栏配置、滚动和标题栏。
9. 工程门禁：前端全部测试、Lint 和生产构建通过。

## 分阶段实施建议

### 第一阶段：设置中心骨架

- 新增设置覆盖层、左侧导航、搜索、返回工作台和视图状态。
- 证明工作台保持挂载，完成入口迁移和齿轮移除。

### 第二阶段：现有配置迁移

- 依次迁入模型、搜索、姬子和工具内容面板。
- 每迁入一页即删除对应旧入口，保持单一事实源。

### 第三阶段：系统与数据

- 汇总后台状态、版本、目录、环境检查和修复指引。
- 完成双分辨率和最小窗口视觉验收。

## 不在本次范围

- Skill 管理、Skill 导入和 Skill 复写迁移。
- Agent、节点、画布、运行历史、报告和诊断配置迁移。
- 清空全部本地数据或批量删除用户资产。
- 账号、云同步、插件市场、主题系统、快捷键系统等当前不存在的能力。
- 手机和平板设置界面。
