# 模块化拆分规范

> 更新日期：2026-07-06
> 目标：让项目保持“主干 + 功能节点”的结构。主干负责调度和接线，功能节点负责自己的类型、状态、执行、展示和工具逻辑。

## 一、核心原则

1. 主入口只做接线，不承载大段业务逻辑。
2. 一个功能优先拥有独立文件夹；功能内部再按 `types`、`helpers`、`executor`、`components`、`hooks` 拆分。
3. 外部调用先走稳定门面文件，避免组件直接依赖深层实现。
4. 删除某个功能时，只允许影响依赖它的功能；不应影响无依赖的并行功能。
5. 状态层只负责状态读写，复杂构造、转换、恢复、校验逻辑放到功能模块。
6. 新增功能必须先确认归属目录；没有归属时，先建功能目录，而不是继续塞进现有大文件。

## 二、当前拆分基线

### Canvas Store

- `app/src/stores/canvasStore.ts`：稳定门面与 Zustand 主干。
- `app/src/stores/canvas/types.ts`：画布、节点、运行状态、历史记录类型。
- `app/src/stores/canvas/naming.ts`：默认命名与输出目录命名。
- `app/src/stores/canvas/selectors.ts`：脏检查、上游节点名等查询逻辑。
- `app/src/stores/canvas/derived.ts`：折叠、隐藏、派生图状态重算。
- `app/src/stores/canvas/runRecovery.ts`：异常退出后的运行态恢复。
- `app/src/stores/canvas/runRecords.ts`：运行快照、运行历史、只读回看、产物删除标记。

### Master Actions

- `app/src/lib/masterActions.ts`：稳定门面，保留旧导入路径。
- `app/src/lib/masterActions/types.ts`：总 Agent 行动类型。
- `app/src/lib/masterActions/textGuards.ts`：确认/取消文本识别。
- `app/src/lib/masterActions/parser.ts`：规则兜底解析。
- `app/src/lib/masterActions/descriptions.ts`：确认卡片与步骤文案。
- `app/src/lib/masterActions/helpers.ts`：Agent 匹配、节点创建、节点查找与连线辅助。
- `app/src/lib/masterActions/planExecutor.ts`：LLM 结构化计划逐步执行。
- `app/src/lib/masterActions/executor.ts`：总执行入口。

### Agent Runner

- `app/src/lib/agentRunner.ts`：运行调度主干，负责创建运行副本、节点并发调度、状态推进、中止与批量画布运行。
- `app/src/lib/agentRunner/types.ts`：运行结果、节点输出、运行产物和中止错误类型。
- `app/src/lib/agentRunner/constants.ts`：并发、重试、摘要等运行参数。
- `app/src/lib/agentRunner/graph.ts`：拓扑排序、入边/出边索引。
- `app/src/lib/agentRunner/inputs.ts`：数据源读取、前序输出拼装、输入 schema 匹配和转换。
- `app/src/lib/agentRunner/modelCalls.ts`：节点 prompt 构造和 LLM 调用。
- `app/src/lib/agentRunner/outputFormats.ts`：输出格式识别和格式元信息。
- `app/src/lib/agentRunner/structuredOutput.ts`：结构化 JSON 解析、内置 schema 校验、摘要和思维导图 HTML。
- `app/src/lib/agentRunner/outputWriter.ts`：Markdown/Word/Excel/思维导图写入和 `data.json` 产物记录。
- `app/src/lib/agentRunner/utils.ts`：运行器通用小工具。

### Master Agent Panel

- `app/src/components/MasterAgentPanel.tsx`：总 Agent 会话流程主干。
- `app/src/components/MasterAgentPanel/MessageList.tsx`：消息列表、欢迎建议和待确认操作卡片。
- `app/src/components/MasterAgentPanel/Composer.tsx`：输入框、附件、模型选择、联网搜索和停止按钮。
- `app/src/components/MasterAgentPanel/fileHelpers.tsx`：附件图标、安全链接、搜索警告和 base64 读取。
- `app/src/components/MasterAgentPanel/summary.ts`：上下文摘要压缩 prompt。
- `app/src/components/MasterAgentPanel/actionCustomization.ts`：确认卡片自定义值逻辑。
- `app/src/components/MasterAgentPanel/types.ts` / `constants.ts`：面板局部类型与常量。

### Canvas And Panels

- `app/src/components/CanvasArea.tsx`：画布 Flow 主干，保留拖拽落点、节点变更和 ReactFlow 接线。
- `app/src/components/CanvasArea/ZoomSlider.tsx`：缩放条。

## 三、新功能落位规则

1. 画布状态相关：放入 `app/src/stores/canvas/<feature>.ts`，再由 `canvasStore.ts` 接线。
2. 总 Agent 操作相关：放入 `app/src/lib/masterActions/<feature>.ts`，再由 `masterActions.ts` 导出。
3. 运行器相关：优先在 `app/src/lib/agentRunner/` 新建子模块，再保留 `agentRunner.ts` 门面。
4. UI 组件超过约 300 行时，应拆成同名目录，例如 `MasterAgentPanel/Composer.tsx`、`MessageList.tsx`、`PendingActionCard.tsx`。
5. 跨功能共享逻辑必须放在 `lib/` 或对应领域的 `shared` 模块，禁止两个功能互相深层导入实现细节。
6. 跨功能共享的 Agent 节点基础工具放在 `app/src/lib/agentNode.ts`，例如 `nodeLabel()`；不要从 `agentRunner/*` 或 `masterActions/*` 互相导入内部实现。

## 四、验收要求

每次拆分必须满足：

- 旧导入路径继续可用。
- 不改变持久化数据结构，除非有明确迁移。
- `npm.cmd run lint` 通过。
- `npm.cmd run build` 通过。
- 涉及 Rust/Python 时补跑对应检查。
