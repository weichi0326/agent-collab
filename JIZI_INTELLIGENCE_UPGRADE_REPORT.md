# 姬子智能升级验证报告

日期：2026-07-11

## 已完成能力

1. 默认提示词已与受控动作能力一致，不再否认画布操作。
2. 故障诊断采用严格分类；解析失败、低置信度或无证据时归类为未知，禁止默认生成工具。
3. 项目观察包含稳定 ID、完整节点与连线、Agent/模型/工具配置、运行状态、输出摘要和失败证据，且不包含 API Key。
4. 新增 Agent/节点配置修改、画布删除、自定义工具覆盖与删除动作。
5. 动作执行前检查对象、模型、工具、只读状态、画布上限和 8 步上限。
6. 画布和 Agent 使用本地快照事务；工具使用后端源码快照和补偿安装恢复。
7. 删除画布和删除工具必须经过两次确认，第一次确认不会调用执行器。
8. 有限自主状态机具备 8 步、单步修正 1 次、重新规划 2 次和连续无变化停止限制。
9. 执行后重新观察项目并确定性验证；验证失败会携带证据重新规划，新写计划再次等待确认。
10. 长期记忆 Store 升级为 v6 结构化记录，旧字符串记忆无损迁移；支持来源、时间、置信度、作用域、覆盖、冲突和过期状态。
11. 深度搜索在后端校验公网地址并限制重定向、超时、类型和体积；正文失败时降级到搜索摘要。
12. 建立 63 项固定智力评测场景，覆盖画布、Agent、工具、恢复、搜索、记忆、改口、取消和回滚。

## 验证证据

- 前端：26 个测试文件，150 项测试通过。
- 前端 Lint：通过，无警告。
- 前端生产构建：通过。
- Python：安装器和网页安全策略 42 项测试通过。
- Python：新增路由与工具模块通过语法编译。
- Rust：9 项测试通过。
- Rust：`cargo check` 通过。
- Rust：`cargo clippy -- -D warnings` 通过。

## 安全边界

- 所有写操作仍需用户确认。
- 删除操作需二次确认。
- 模型不能直接修改 Store 或文件。
- 内置工具不能覆盖或删除。
- 工具快照读取仅允许注册表内自定义工具，并拒绝不安全模块路径。
- 网页正文读取拒绝非 HTTP(S)、凭据 URL、localhost、私网、回环和链路本地地址。

## 残余限制

1. 63 项评测场景是固定语料与结构门槛，不代表已经用所有外部模型逐项跑出真实成功率；模型相关成功率需要在指定模型和密钥环境中单独记录。
2. 网页来源等级目前采用域名规则，关键事实交叉核对仍由回答模型执行，不是独立事实图谱。
3. 工具依赖包缓存不会在回滚时卸载；回滚保证原工具代码、元数据和可调用状态恢复。
4. DNS 校验覆盖每个请求和重定向目标，但底层 HTTP 客户端仍存在 DNS 校验到连接之间的理论重绑定窗口，后续可通过固定已验证 IP 的传输适配器进一步收紧。

## 当前判断

姬子已经从一次性固定动作助理升级为受控的有限自主 Agent：能够观察、规划、确认、事务执行、验证和在失败时重新规划。结构性智能预计约为 4.3/10，强模型下体感可接近 5/10；尚未达到 Codex 级通用工程智能体水平。
