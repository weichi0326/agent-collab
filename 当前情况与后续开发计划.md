# Agent Collab 当前情况与后续开发实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use `executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. 当前仓库不要求启用子 Agent；除非用户明确要求，不要主动扩大并行开发范围。

**目标：** 让接手者无需回看历史对话即可理解当前软件能力、本次 Agent 节点改造、验证状态、已知阻塞与下一阶段优先级。

**架构：** 软件保持“一种 Agent 节点 + 属性面板组合能力”的设计，不拆分专用节点类型。画布节点配置保存在 `AgentNodeData.capabilities`，运行时由纯配置归一函数解释，再分别进入输入组装、模型调用、执行策略和输出校验链路。

**技术栈：** Tauri 2、React 19、TypeScript 6、Vite 8、Zustand、Ant Design 6、XYFlow、Python FastAPI。

## 全局约束

- 仅面向 Windows 桌面端，应用版本当前为 `3.1.0`。
- 保持一种 Agent 节点，通过开关和折叠区域组合能力。
- 未配置 `capabilities` 的节点必须维持原有运行行为。
- 网页 URL 输入暂不实现抓取，只显示“暂未支持”并禁止选择。
- 暂不加入批处理、多产物、缓存、人工审批、多候选和模型评审器。
- 不修改或提交本地生成目录 `.superpowers/`。
- 不提交 `data/` 内的模型、搜索配置和任何 API Key。
- `.bat` 文件保持 GBK + CRLF，不用 UTF-8 重写。
- 任何新行为先写失败测试，再写最小实现；提交前至少运行前端测试、lint、Python 测试和构建。

### 3. 本轮完成的 Agent 节点能力

#### 输入处理

- 支持全部上游或指定上游。
- 支持手动调整指定上游处理顺序。
- 支持兼容、智能、结构化、摘要、完整正文五种内容模式。
- 智能模式按“结构化数据 > 摘要 > 完整正文”选择一种内容，避免重复注入。
- 有上游时可额外读取文件或历史产物。
- 支持输入字符上限和报错、截断、模型压缩三种超长策略。
- URL 入口保留但禁用；运行层遇到旧 URL 配置会明确报“暂未支持”。

#### 模型生成

- `max_tokens` 支持 `512-16384`，默认 `4096`。
- 温度支持继承或 `0-2` 自定义。
- 支持回退模型；主模型及其重试耗尽后，回退模型执行一次。
- 支持空输出自动重试。
- Python `llm-calling` 已接收并向 OpenAI 兼容协议和 Anthropic 传递温度。
- Python 服务契约版本已更新为 `2026-07-21.agent-node-options`。

#### 执行策略

- 节点重试次数支持 `0-2`。
- 单次节点超时支持 `30-300` 秒。
- 超时转换为可重试的节点错误，不冒充用户主动取消。
- 运行快照中的失败节点可按配置手动重跑自身及下游，成功上游不会重复执行。

#### 质量校验

- 支持最少/最多字符。
- 支持必含词和禁用词。
- 支持失败停止或按节点策略重试。
- 现有自定义 Schema、内置文件结构校验错误也会遵循“失败重试”。
- Schema 配置本身不是合法 JSON 时不会无意义重试。

#### 配置兼容

- 新能力统一保存在 `AgentNodeData.capabilities`。
- 能力关闭或字段缺失时使用旧默认行为。
- 画布导出/导入会保留能力配置。
- 导入时会重新映射 `selectedUpstreamIds` 和 `upstreamOrder`。
- 回退模型的 `configId` 属于本机配置，跨机器导出时主动清空。

### 4. 主要代码位置

- `app/src/stores/canvas/types.ts`：节点能力类型。
- `app/src/lib/agentNodeCapabilities.ts`：配置归一、输入长度策略、尝试计划和文本校验。
- `app/src/lib/agentRunner/inputs.ts`：上游选择、内容模式、补充来源和超长输入处理。
- `app/src/lib/agentRunner/modelCalls.ts`：节点生成参数和模型覆盖。
- `app/src/lib/agentRunner/execution.ts`：超时、校验错误和重试分类。
- `app/src/lib/agentRunner.ts`：主模型重试、回退模型、校验和落盘执行顺序。
- `app/src/components/PropertiesPanel.tsx`：四组折叠能力 UI。
- `app/src/components/PropertiesPanel/NodeRunStatus.tsx`：失败节点手动重跑入口。
- `app/src/lib/canvasTransfer.ts`：能力配置导入导出。
- `python/tools/llm_calling.py`：`temperature` 和 `max_tokens` 后端参数。
