What it does

一个工具，覆盖三类典型场景

从单点任务到多步流水线，把"人工切流程"变成"Agent 自跑"。

游戏测试流程自动化

分析策划需求 → 提取测试点 → 生成测试用例 Excel → 跑测后产出 Bug 报告。一整条流水线在一个画布里跑完。

文档处理与报告生成

读取 .txt / .md / .docx / .pdf，按模板生成 Word/Excel/思维导图/Markdown 报告，同步落盘机器可读 JSON 供下游消费。

多步 LLM 任务编排

把一次大任务拆给多个 Agent：每个节点选自己的模型、提示词、工具标签，前序输出自动流转到下游，schema 不匹配还能 LLM 兜底转换。

Core Features

已落地的核心功能

从画布编排到执行调度、从工具调用到报告管理，每一环都已跑通冒烟验证。

可视化画布编排

左侧 Agent 库拖入画布、画箭头连线即建流水线。支持多画布（最多 6 个并行）、撤销、框选、复制粘贴、子节点递归折叠，所有交互含快捷键。

React Flow 最多 6 画布并行 Ctrl+Z 撤销 框选/复制粘贴

两层 Agent 模型

Agent 库编辑「定义」，拖到画布即「快照复制」为节点实例，定义与实例完全解耦。支持克隆、删除前引用检查、拖拽排序与 3 个预置模板。

定义 vs 实例 克隆 引用检查 拖拽排序

总 Agent「姬子」

顶部抽屉式面板，用自然语言就能创建画布/Agent/节点/连线、改输出格式、运行当前画布。LLM 结构化规划 + 多步确认 + 规则兜底，多会话独立消息流。

LLM 结构化规划 多会话 多步确认 规则兜底

多 LLM 配置中心

预置 13 家厂商（OpenAI / Anthropic / DeepSeek / Kimi / MIMO 等），同厂商多份配置、密钥明文存储、精选星标置顶、通信测试延迟分档、能力标签（1M/视觉/语音）。

13 家预置 通信测试 能力标签 会话隔离

依赖图工作流引擎

单画布依赖图调度：顺序执行、依赖满足后并行（最大 4 节点并发、80ms 错峰）、失败节点只跳过自己后续依赖、临时 LLM 失败节点级补跑、多画布批量并行。

顺序/并行 失败隔离 节点级重试 多画布并行

双格式产物 + Schema 校验

每个节点同时输出人类可读主产物（Markdown / Word / Excel / 思维导图）和机器可读 data.json 信封，内置 schema 校验 + 一次自动修复，下游优先消费结构化 JSON。

4 种产物格式 data.json 信封 Schema 校验 LLM 自动修复

6 个内置 Python 工具

file-read / file-write / excel / pdf-read / docx-read / llm-calling，全部走本地 FastAPI 服务（127.0.0.1:18081）。路径沙箱 + 大小限制 + SSRF 防护，Tauri 启动自动拉起。

FastAPI 本地服务 路径沙箱 SSRF 防护 自动拉起

联网搜索 + 记忆系统

多厂商故障转移（Tavily → Brave → Serper），逐家测试 Key 即时验证；记忆系统按关键词检索用户画像/偏好/资源并注入系统提示词，搜索结果以参考来源附在 AI 回复下方。

3 家搜索厂商 故障转移 参考来源 关键词记忆

报告中心

集中展示总运行次数、成功率、产物数、平均耗时、失败/中止数、最近产出、画布/节点表现；历史产物表格支持按画布/格式/关键词筛选、定位文件夹、二次确认删除。

运行指标 画布/节点表现 筛选表格 分级 outputs/

运行历史 + 只读回看

每次运行自动生成只读历史画布 tab，同屏回看节点状态与产物；中止后产物可保留或移除；应用崩溃重启后会自动恢复遗留运行态，避免僵尸任务锁定页签。

How it works

三步搭一条 Agent 流水线

从拖入节点到拿到产物，整条链路无需写一行代码。

① 拖入并连线

从左侧 Agent 库拖节点到画布，画箭头连出数据流向

② 点运行 / 让姬子跑

标题栏点运行，或让姬子用自然语言多步执行

③ 拿到本地产物

主产物 + data.json 同步落盘到 outputs/，可双击打开

Highlights

为什么值得用

不是又一个 LLM 聊天框，而是把 Agent 真正串成可重复运行的工作流。

01

可视化 ≠ 写代码

拖拽 + 连线即完成编排，比写脚本快、比纯聊天准。运行副本机制让原画布不被运行状态污染。

02

依赖图自动调度

前序节点成功后下游自动并行；某节点失败只跳过它的后续依赖，旁支照常跑完，整体不中断。

03

姬子一句话搞定

"建一条需求→用例→Bug 的流水线" —— 姬子直接 LLM 规划出多步动作、列表确认后执行，写不下脚本的工作流也能说出口。

04

多 LLM 自由混搭

同一画布里不同节点可挂不同厂商模型，会话严格隔离；联网搜索三家故障转移，Key 还能逐家测。

05

双格式落盘可流转

人看 Markdown/Word/Excel/思维导图，机器读 data.json；下游优先吃结构化 JSON，schema 不匹配还能 LLM 兜底转换。

06

数据便携、本地优先

所有配置与产物落项目内明文 JSON / outputs/，Tauri Rust 命令读写，dev/release 数据根分明，不上云、可整包迁移。
